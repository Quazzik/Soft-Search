# Примитивный сервис аналитики продаж
Написаный почти без вмешательства человека, на бесплатном инструменте
Для запуска достаточно node.js и *npm run dev* в IDE, но лучше всего SSG ощущется если собрать проект через *npm run build*, а затем запустить через *npm run start*.

## Это вайб-проект на один вечер, в нём ии-агент разработал платформу аналитики
**Сам проект создавался для достижения нескольких целей**:
1. Повторить и попробовать новые идеи в проекте реализуемом на работе
2. Изучить мягкий поиск через Fuse.js
3. Познакомиться со стратегией рендера Static Site Generation
4. Потрогать зависящий от времени refetch
5. Занять вечернее время полезным делом

Если я к нему всё-таки вернусь, то в планах расширение проекта и приблиэение его к проду:
1. Создание отдельного АПИ с которым будет общаться приложение
2. Создание базы данных из которой приложение будет брать информацию
3. Обмазывание всего приложения метриками и логами
4. Создание дашборда с отображением метрик
5. Контейнеризация

Если даже эти цели будут достигнуты, то последние запланированные изменения:
1. Подключение кэш-базы по типу Clickhouse или Redis
2. Подключение Kafka для работы с логами и метриками
3. Backend по таймеру обновляет данные в Clickhouse, публикует diff данных в kafka, web запрашивает данные и ререндерится только узнав о том что в базе есть что-то новое
